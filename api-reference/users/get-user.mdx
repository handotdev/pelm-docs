---
title: "Get User"
description: "Get details for a `User`."
api: "GET /users/{user_id}"
---

### Path Params

<ParamField path="user_id" type="string" required>
  `user_id` for the user that you submitted when you first created a Connect Token for
  this user
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  `access_token` passed as "Bearer \{access_token}"
</ParamField>

<ParamField header="Pelm-Client-Id" type="string" required>
  `Pelm-Client-Id` given on registration
</ParamField>

<ParamField header="Pelm-Secret" type="string" required>
  `Pelm-Secret` given on registration
</ParamField>

<RequestExample>

```bash cURL
curl --request GET \
     --url https://api.pelm.com/users/user_id \
     --header 'accept: application/json'
```

</RequestExample>

### Response

<ResponseField name="id" type="string">
  The id you specified when you initially created the Connect Token
  to connect this user's utility account
</ResponseField>

<ResponseField name="utility" type="object">
  <Expandable>
    <ResponseField name="id" type="number"></ResponseField>
    <ResponseField name="full_name" type="string"></ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="utility_username" type="string">
  The id you specified when you initially created the Connect Token
  to connect this user's utility account
</ResponseField>

<ResponseField name="credentials_valid" type="boolean">

A boolean field indicating whether the credentials we have for this `User` are
valid

A common scenario that would cause this field to be false is your
user changing their password. If this field is false, then we're unable to
pull the latest data for this `User` and your user needs to go through the
Connect flow again.

</ResponseField>
