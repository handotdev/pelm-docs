---
title: "Create Access Token"
api: "POST /auth/token"
contentType: "application/x-www-form-urlencoded"
---

Exchange an `authorization_code` for an `access-token`. The `authorization_code` is returned in the Connect `onSuccess` callback after the user successfully submits their utility credentials.

The `access_token` will not expire. Make sure you securely store it in your database so you can make future requests for the given user.

The fields `access_token_expires_in`, `refresh_token`, and `refresh_token_expires_in` can be ignored. These are included for applications using our legacy authorization framework.

### Form data

<ParamField body="code" type="string" required>
  authorization code from Connect.
</ParamField>

### Headers

<ParamField header="Pelm-Client-Id" type="string" required>
  Pelm-Client-Id given to a developer when they register
</ParamField>

<ParamField header="Pelm-Secret" type="string" required>
  Pelm-Secret given to a developer when they register
</ParamField>

<RequestExample>

```bash cURL
curl --request POST \
     --url https://api.pelm.com/auth/token \
     --header 'accept: application/json' \
     --header 'content-type: application/x-www-form-urlencoded'
```

</RequestExample>

### Response

<ResponseField name="access_token" type="string"></ResponseField>

<ResponseField name="access_token_expires_in" type="string">
  number of seconds until access_token expires (1 hour)
</ResponseField>

<ResponseField name="refresh_token" type="string"></ResponseField>

<ResponseField name="refresh_token_expires_in" type="string">
  number of seconds until refresh_token expires (1 year)
</ResponseField>
